# AI编辑助手 · 单机永久部署教程

本教程适用于：**只在一台 Windows 电脑上使用**，不再依赖 `npm start` 开发模式，开机后启动一个后台服务即可在 Word 中使用插件。

---

## 一、部署原理（先看懂再操作）

```
┌─────────────┐      HTTPS       ┌──────────────────────────────┐
│  Word 桌面版 │ ──────────────► │ https://localhost:3000       │
│  加载插件    │                  │  server/standalone.js        │
└─────────────┘                  │  ├─ 静态页面 (dist/)         │
                                 │  └─ /api/* 代理大模型请求    │
                                 └──────────────────────────────┘
```

与开发模式的区别：

| 项目 | 开发模式 `npm start` | 单机生产模式 |
|------|---------------------|--------------|
| 前端 | webpack 热更新 dev-server | 预构建的 `dist/` 静态文件 |
| 代理 | 独立进程 `:3001` | 合并到 `:3000` 同一服务 |
| 是否需开终端 | 必须保持 `npm start` | 只需 `npm run start:prod`（可注册开机自启） |
| Word 安装 | 每次旁加载 | **一次性**上传 manifest |

---

## 二、环境要求

| 项目 | 要求 |
|------|------|
| 系统 | Windows 10 / 11 |
| Word | 2016 或更高，推荐 Microsoft 365 桌面版 |
| Node.js | 18.x 或 20.x LTS |
| 网络 | 使用 AI 功能时需能访问对应大模型 API |

## 三、完整安装步骤

### 步骤 1：安装项目依赖

以 **PowerShell** 打开终端：

```powershell
cd "E:\2-AIprogram\2-word插件\AI编辑助手"
npm install
```

> 若路径不同，请替换为你本机的实际项目路径。

---

### 步骤 2：构建生产包

```powershell
npm run build
```

成功后会在项目下生成 `dist/` 目录：

```
dist/
├── manifest.xml       ← Word 加载项清单
├── taskpane.html      ← 侧栏页面
├── taskpane.js        ← 侧栏逻辑
├── commands.html
├── commands.js
└── assets/            ← 图标
```

验证构建：

```powershell
npm run validate
```

应看到 **The manifest is valid.**

---

### 步骤 3：安装本机 HTTPS 证书

Office 加载项**必须使用 HTTPS**。单机部署复用开发证书（信任 `localhost`）：

```powershell
npx office-addin-dev-certs install
```

弹出提示时选择 **是 / 允许**。

若 Word 功能区图标显示为空白，以**管理员身份**打开 PowerShell 执行：

```powershell
npm run install-certs
```

---

### 步骤 4：将插件注册到 Word（一次性）

```powershell
npm run install:local
```

该命令会自动完成：
- 生成图标
- 安装/更新 HTTPS 证书
- 配置 localhost 回环权限
- 将 manifest 注册到 Word

---

### 步骤 5：启动单机生产服务

```powershell
npm run start:prod
```

看到以下输出表示成功：

```
[prod] https://localhost:3000 (v5)
[prod] 单机生产服务已启动（静态页面 + API 代理）
```

**保持此终端窗口运行**（或按第六节配置开机自启）。

验证：用浏览器打开 `https://localhost:3000/taskpane.html`，应能看到插件页面（可能有证书提示，点继续访问即可）。

---

### 步骤 6：在 Word 中添加加载项（一次性）

1. **完全退出 Word**（任务管理器中确认无 `WINWORD.EXE` 进程）
2. 重新打开 Word
3. 进入 **插入** → **我的加载项** → **上传我的加载项**
4. 选择文件：
   ```
   E:\2-AIprogram\2-word插件\AI编辑助手\dist\manifest.xml
   ```
5. 点击 **上传**

> 若「上传我的加载项」不可用，可用备用方式：
> - **文件** → **选项** → **信任中心** → **信任中心设置** → **受信任的加载项目录**
> - 添加 `dist` 文件夹路径，勾选「在菜单中显示」，重启 Word

---

### 步骤 7：开启 Word 连接体验（重要）

若侧栏报「所需的服务已关闭」：

1. Word → **文件** → **选项**
2. 左下角 **隐私设置**（或 **文件 → 帐户 → 管理设置**）
3. 勾选：
   - **开启所有已连接的体验**
   - **启用下载在线内容的体验**
4. 确定后**重启 Word**

---

### 步骤 8：验证插件可用

1. 打开 Word **开始** 选项卡
2. 找到 **AI编辑助手** 按钮组，点击打开侧栏
3. 点击侧栏右上角 **设置**，填写至少一个 API Key
4. 在文档中选中一段文字，测试「汇总」「精简」等功能

## 四、日常使用方式

每次使用 Word 前，确保生产服务在运行：

```powershell
cd "E:\2-AIprogram\2-word插件\AI编辑助手"
npm run start:prod
```

然后正常打开 Word 即可，**不需要**再执行 `npm start` 或 `npm run sideload`。

## 五、开机自动启动（推荐）

避免每次手动开终端，可用以下任一方式。

### 方式 A：任务计划程序（无需额外安装）

1. 按 `Win + R`，输入 `taskschd.msc`，回车
2. **创建基本任务**
   - 名称：`AI编辑助手生产服务`
   - 触发器：**当用户登录时**
   - 操作：**启动程序**
   - 程序：`node`
   - 参数：`server\standalone.js`
   - 起始于：`E:\2-AIprogram\2-word插件\AI编辑助手`
3. 完成创建
4. 在任务属性中勾选 **「使用最高权限运行」**（可选，用于证书相关场景）
5. 注销或重启后，服务会自动在后台运行

### 方式 B：PM2 进程管理器

```powershell
npm install -g pm2
cd "E:\2-AIprogram\2-word插件\AI编辑助手"
pm2 start server/standalone.js --name ai-editor
pm2 save
pm2 startup
```

按 PM2 提示执行它给出的命令，即可开机自启。

查看状态：

```powershell
pm2 status
pm2 logs ai-editor
```

## 六、更新插件版本

代码有更新时，按以下顺序操作：

```powershell
# 1. 停止旧服务（Ctrl+C 或 pm2 stop ai-editor）

# 2. 重新构建
npm run build

# 3. 重启服务
npm run start:prod
# 或 pm2 restart ai-editor

# 4. 在 Word 侧栏按 Ctrl+F5 强制刷新
```

若 manifest 有变更，重新执行 `npm run install:local` 并在 Word 中重新上传 manifest。

## 七、卸载

### 7.1 从 Word 移除插件

1. **插入** → **我的加载项**
2. 找到 **AI编辑助手** → 右键 **删除**
3. 重启 Word

### 7.2 停止后台服务

- 手动模式：关闭运行 `start:prod` 的终端
- PM2 模式：`pm2 delete ai-editor`
- 任务计划：在任务计划程序中禁用或删除对应任务

### 7.3 清除注册信息（可选）

删除注册表项（`regedit`）：

```
HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\WEF\Developer
```

删除与 AI编辑助手 相关的字符串值。

## 八、常见问题

### Q1：Word 看不到「AI编辑助手」按钮

| 检查项 | 处理 |
|--------|------|
| 生产服务未启动 | 执行 `npm run start:prod` |
| manifest 未上传 | 重新执行步骤 6 |
| Word 未重启 | 完全退出 Word 后重开 |
| 连接体验未开启 | 见步骤 7 |

### Q2：侧栏空白或加载失败

```powershell
# 确认服务运行
npm run start:prod

# 浏览器测试
# 打开 https://localhost:3000/taskpane.html

# 重装证书
npx office-addin-dev-certs install

# 清除 Office 加载项缓存后重启 Word
# 删除文件夹: %LOCALAPPDATA%\Microsoft\Office\16.0\Wef\
```

### Q3：AI 功能报错

| 现象 | 处理 |
|------|------|
| 未提供 API Key | 在侧栏设置中填写并保存 |
| 连接超时 | 检查网络，确认能访问大模型 API |
| 端口被占用 | 关闭 `npm start` 或其他占用 3000 端口的程序 |

### Q4：与开发模式冲突

开发模式（`npm start`）和单机模式（`start:prod`）**不能同时运行**，都占用 3000 端口。

切换前请先关闭另一个：

```powershell
# 停止开发模式：在 npm start 终端按 Ctrl+C
# 或
npm run stop
```

## 十、与 COM 加载项的区别

本插件是 **Office Web 加载项**，不是传统的 **COM 加载项**（文件 → 选项 → 加载项 → COM 加载项）。Web 加载项是微软目前推荐的方式，无需重写代码，通过 manifest + HTTPS 页面即可在 Word 中运行。
