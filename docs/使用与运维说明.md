## 1. 架构概览

```
┌─────────────────────────────────────────────────────────────┐
│  你的电脑（开发）                                            │
│  npm start → localhost:3000（页面）+ localhost:3001（API）   │
└──────────────────────────┬──────────────────────────────────┘
                           │ git push
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  GitHub → Vercel 自动构建部署                                │
│  https://milei.dpdns.org                                    │
│    ├── /taskpane.html   侧栏页面                            │
│    ├── /manifest.xml    插件清单                            │
│    └── /api/*           AI 代理接口                         │
└──────────────────────────┬──────────────────────────────────┘
                           │ manifest 指向线上地址
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  朋友 / 你的 Word                                            │
│  开始 → AI编辑助手 → 从 milei.dpdns.org 加载侧栏             │
│  API Key、代理令牌 → 保存在本机 Word 存储                    │
└─────────────────────────────────────────────────────────────┘
```

**核心原则：**

- `manifest.xml` = 安装时一次性注册的「地址本」，告诉 Word 从哪加载插件
- 页面和 API 代码在 Vercel 上，**更新代码后朋友自动用到新版**（无需重装）
- API Key 和代理令牌存在 Word 本机，**与 manifest 路径无关**（但与域名有关）

## 2. 线上环境（给朋友用）

### 2.1 验证服务是否正常

浏览器访问：

| 地址 | 期望结果 |
|------|----------|
| `https://milei.dpdns.org/api/health` | `{"status":"ok",...}` |
| `https://milei.dpdns.org/taskpane.html` | 侧栏页面 |
| `https://milei.dpdns.org/manifest.xml` | URL 均为 `milei.dpdns.org` |
| `https://milei.dpdns.org/` | 说明首页（非 404） |

### 2.2 Vercel 环境变量（Production）

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `ADDIN_BASE_URL` | `https://milei.dpdns.org` | 构建时写入 manifest 的域名 |
| `PROXY_ACCESS_TOKEN` | 随机长字符串 | API 代理访问控制，与 Word 设置一致 |

修改环境变量后必须 **Redeploy** 才生效。

> Vercel 编辑环境变量时 Value 显示为空是正常现象（安全隐藏），不代表未保存。

### 2.3 自定义域名 DNS（Cloudflare）

| 类型 | 名称 | 值 |
|------|------|-----|
| A | `@` | `216.198.79.1`（以 Vercel 显示为准） |
| TXT | `_vercel` | 域名验证记录（验证通过后可删除） |

A 记录代理状态建议 **DNS only（灰云）**。

## 3. 分发给朋友安装

### 3.1 安装包位置

```
E:\2-AIprogram\2-word插件\AI编辑助手\AI编辑助手-安装包\
E:\2-AIprogram\2-word插件\AI编辑助手\AI编辑助手-安装包.zip
```

### 3.2 打包前你要做

1. 编辑 `安装令牌.txt`，填入你的 `PROXY_ACCESS_TOKEN`
2. 确认 `https://milei.dpdns.org/api/health` 正常
3. 压缩 `AI编辑助手-安装包` 文件夹发给朋友
4. **勿公开发布**安装包（含代理令牌）

### 3.3 朋友安装步骤（3 步）

```
1. 解压 → 双击「安装.bat」→ 看到 Done
2. 完全退出 Word → 重新打开 → 开始 → AI编辑助手
3. 设置 → 填代理令牌 + 自己的 DeepSeek API Key → 保存
```

### 3.4 重要说明

| 误区 | 正确做法 |
|------|----------|
| 插入 → 上传我的加载项 | 很多 Word 中文版**没有此项**，用 `安装.bat` |
| 开始 → 加载项（商店列表） | 插件入口在 **开始 → 右侧 AI编辑助手 按钮** |
| 需要安装 Node.js | 朋友**不需要**，一键脚本已处理 |

### 3.5 朋友安装后的文件位置

```
C:\Users\用户名\Documents\AI-Word-Editor\manifest.xml
```

## 4. 朋友安装后如何自动更新

你更新功能并 `git push` 后，Vercel 自动部署。朋友**无需重装**，下次打开侧栏即加载新代码。

| 你的操作 | 朋友需要做什么 |
|----------|----------------|
| 改 UI、功能、修 bug | 通常**不用做任何事** |
| 界面还是旧的 | 侧栏按 `Ctrl + F5` 强制刷新 |
| 改了 API 服务端 | 自动生效 |
| 换了域名 | 需重新发安装包 |
| 改了 manifest（权限/按钮/插件 ID） | 需重新运行 `安装.bat` |

## 5. 本地开发调试

线上和朋友用的版本**不受**本地开发影响，可以同时存在。

### 5.1 启动本地服务

```powershell
cd "E:\2-AIprogram\2-word插件\AI编辑助手"
npm start
```

| 服务 | 地址 | 作用 |
|------|------|------|
| 前端 | `https://localhost:3000` | 侧栏页面（热更新） |
| API 代理 | `http://localhost:3001` | 转发大模型请求 |

### 5.2 注册本地 manifest

```powershell
npx office-addin-dev-settings register "E:\2-AIprogram\2-word插件\AI编辑助手\manifest.xml"
```

完全退出 Word 后重新打开 → **开始 → AI编辑助手**。

### 5.3 本地开发配置

- **代理访问令牌**：可留空（本地代理不校验 `PROXY_ACCESS_TOKEN`）
- **API Key**：需在设置中填写（若换过域名/环境，可能需重新填一次）

### 5.4 仅预览 UI（不用 Word）

浏览器打开 `https://localhost:3000/taskpane.html` 可看界面，但无法使用 Word 选区相关功能。

## 6. 开发环境与线上环境切换

Word 同一时刻只加载**一个** manifest 指向的地址。

| 目的 | 注册的 manifest | 是否需要 npm start |
|------|-----------------|-------------------|
| **本地调试** | `AI编辑助手\manifest.xml`（localhost:3000） | 需要 |
| **使用线上版** | `2-word插件\manifest.xml`（milei.dpdns.org） | 不需要 |

### 切换到本地开发

**快捷方式：双击项目根目录 `切换到本地开发.bat`**

或手动执行：

```powershell
cd "E:\2-AIprogram\2-word插件\AI编辑助手"
npm start
# 另开终端
npx office-addin-dev-settings register "E:\2-AIprogram\2-word插件\AI编辑助手\manifest.xml"
# 重启 Word
```

### 切换回线上版本

```powershell
npx office-addin-dev-settings register "E:\2-AIprogram\2-word插件\manifest.xml"
# 或运行 AI编辑助手-安装包\安装.bat
# 重启 Word
```

### 检查当前注册的是哪个 manifest

```powershell
npx office-addin-dev-settings registered
```

## 8. 环境变量说明

### 8.1 Vercel（服务端）

| 变量 | 用途 |
|------|------|
| `ADDIN_BASE_URL` | 构建 manifest 时替换 localhost 为生产域名 |
| `PROXY_ACCESS_TOKEN` | 防止他人滥用你的 API 代理 |

### 8.2 Word 插件设置（本机）

| 设置项 | 用途 |
|--------|------|
| 代理访问令牌 | 与 Vercel `PROXY_ACCESS_TOKEN` 一致 |
| DeepSeek / OpenAI / 通义 API Key | 各平台大模型密钥，仅存本机 |

> 设置按**插件域名**隔离。从 `vercel.app` 切到 `milei.dpdns.org` 后需重新填写一次。

## 10. 常见问题

### Q：朋友 Word 里没有「上传我的加载项」？

正常。使用安装包里的 `安装.bat`，不要走 Word 上传流程。

### Q：安装.bat 窗口乱码？

已改为英文提示，不影响安装。中文说明见 `install-guide.txt`。看到 `Done` 即成功。

### Q：提示「无效的代理访问令牌」？

Word 里的令牌与 Vercel `PROXY_ACCESS_TOKEN` 不一致，或未点「保存设置」。

### Q：manifest.xml 里 URL 变成乱码域名？

`ADDIN_BASE_URL` 填错了（误填部署 ID）。应填 `https://milei.dpdns.org`，然后 Redeploy。

### Q：换域名后 API 配置没了？

设置按域名存储，换域名后需重新填一次代理令牌和 API Key。

### Q：侧栏空白？

1. 检查对应环境是否可访问（本地需 `npm start`）
2. 侧栏 `Ctrl + F5` 强制刷新
3. Word 隐私设置开启「已连接的体验」

### Q：根域名 404？

`/` 需要 `index.html`，已通过构建产物提供。`git push` 部署后 `https://milei.dpdns.org/` 显示说明页。
